api: La carpeta api generalmente se utiliza para alojar archivos relacionados con la API de tu aplicación. 
Aquí es donde puedes definir y organizar tus endpoints de API, es decir, las rutas que los clientes pueden acceder para interactuar con tu aplicación. 
Dentro de esta carpeta, podrías tener archivos que definen las rutas de la API, los controladores asociados con esas rutas, y cualquier lógica adicional necesaria 
para procesar las solicitudes de la API y devolver respuestas adecuadas.


controllers: En la carpeta controllers se encuentran los archivos que contienen la lógica de controladores de tu aplicación.
 Los controladores son responsables de manejar las solicitudes HTTP recibidas desde la API o el frontend, interactuar con los modelos de datos
 para recuperar o modificar información, y devolver una respuesta al cliente. Cada controlador podría manejar un conjunto específico de rutas o endpoints de la API.


models: La carpeta models es donde se almacenan los archivos que contienen los modelos de datos de tu aplicación. 
Estos modelos representan la estructura y las relaciones de los datos en tu base de datos o en otras fuentes de datos. 
Cada modelo generalmente corresponde a una tabla en la base de datos y define los campos y métodos asociados con esos datos. 
Los modelos pueden utilizarse para realizar consultas a la base de datos,
 validar datos antes de almacenarlos, y realizar otras operaciones relacionadas con los datos.


routes: En la carpeta routes se encuentran los archivos que definen las rutas y endpoints de tu aplicación. 
Estos archivos pueden contener la lógica de enrutamiento que mapea las solicitudes HTTP a los controladores correspondientes. 
Los archivos de rutas pueden estar organizados de diversas formas, como por módulo,
 funcionalidad o recurso, para mantener una estructura lógica y modular en tu aplicación.
